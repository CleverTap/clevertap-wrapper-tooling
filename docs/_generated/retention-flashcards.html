The domain (CleverTap SDKs)
What is a native SDK?

The "real" CleverTap library for one platform: clevertap-android-sdk (Kotlin/Java) and clevertap-ios-sdk (Objective-C/Swift). Native apps use it directly.

What is a wrapper SDK?

A thin library that lets cross-platform apps (JavaScript for Cordova/React Native, Dart for Flutter) call the native SDK underneath. CleverTap has clevertap-cordova, clevertap-react-native, clevertap-flutter.

What is the bridge?

The cross-language glue inside a wrapper. When JavaScript calls recordEvent, the bridge carries that call down to the native Android/iOS method that actually does it.

What is the API surface?

The set of public methods the SDK exposes for others to call (e.g. recordEvent, onUserLogin). "Surfacing an API" = exposing a new native method through the wrapper.

What is a module in a native SDK?

A versioned, separately-tagged piece of a native SDK. Android has core, pushtemplates, hms; iOS has core, pushtemplates.

What is a native release / version pin?

The exact native SDK version a wrapper depends on (e.g. the Cordova plugin pins clevertap-android-sdk:8.2.0). "Bumping the pin" = pointing the wrapper at a newer native version.

What is libVersion and how is it computed?

A version encoded as a plain integer in bridge code: major*10000 + minor*100 + patch. Example: 5.2.3 → 50203. It tells the native SDK which wrapper version is calling it.

What is SemVer, and what do MAJOR / MINOR / PATCH mean?

Semantic versioning: MAJOR.MINOR.PATCH. MAJOR = breaking change, MINOR = new backward-compatible features, PATCH = fixes only.

The triage vocabulary
Surfaced — what does it mean?

A native API change the AI implemented in the wrapper (added/updated the bridge).

Skipped — what does it mean?

A change the AI deliberately did not expose (e.g. an internal-only helper).

Deferred — what does it mean?

A change the AI couldn't safely auto-apply, so it left it for a human with a reason.

Flagged for review — and how does it differ from deferred?

A specific deferral: the AI couldn't confirm something (e.g. a method named in the changelog it couldn't find in source), so it raises a flag instead of guessing. In the output JSON its type is removal | deprecation | signature | behavior | unconfirmed.

What is source verification, and why is it non-negotiable?

The rule that the AI must find the exact method/selector in the real native source before writing a call to it. Guessing a symbol that doesn't exist = a broken build. "Verify or flag — never guess."

What is the recall pass?

A second sweep where the AI cross-checks the native changelog against the diff tool's output, to catch the ~20% of changes the regex parser misses.

Precision vs. recall — what's the difference, and who provides each?

Precision = "of what I reported, how much is correct?" (the regex diff — only reports what it parsed). Recall = "of all that truly changed, how much did I find?" (the changelog pass boosts this to catch the regex's ~20% miss).

Python (the diff tool is written in it)
What is Python, and how do you run a file?

A programming language; files end in .py. Run one with python3 file.py. No compile step.

What is the standard library, and why does it matter here?

The batteries included with Python itself. The diff tool uses only the stdlib, so there's nothing to install — no pip install needed.

What is pip?

Python's package installer (an app store for libraries). Not needed for the diff tool.

What is a regex, and what does the diff tool use it for?

A mini-language for text patterns. The diff tool uses regex to find method declarations in source without fully parsing the code. Powerful but imperfect — hence ~80% coverage.

What is an AST, and why does the tool NOT use one?

Abstract syntax tree — the "proper" full-grammar way to parse code. The tool deliberately uses regex instead because it's simpler and good enough (the "regex-based, not AST" design note).

What is a dataclass?

A Python shorthand for a small data-holding object; @dataclass auto-writes the boilerplate. Symbol and Surface in the diff tool are dataclasses.

What does frozen=True do on a dataclass?

Makes it immutable (can't change after creation). Immutable objects can be dict keys and set members — which is how the tool dedupes/compares symbols.

What is a Python set, and what do - and & do?

A collection of unique items. A - B = in A but not B (difference); A & B = in both (intersection); A | B = in either (union). The diff is literally new - old (added) and old - new (removed).

What is a dict, and what's a tuple key?

A lookup table mapping keys to values. A tuple key like ("android","core") lets the tool look things up by platform + module together.

What is tomllib used for?

A stdlib module (Python ≥ 3.11) that reads .toml files. Used to read Android's gradle/libs.versions.toml dependency-versions list.

What is pathlib / Path?

Stdlib way to build file paths with / like Lego (Path.home() / ".cache") instead of gluing strings. Cross-OS.

What is subprocess, and how does the tool "use git"?

Stdlib way to run another command-line program. subprocess.check_output(["git", ...]) literally runs git and captures its output — that's how the Python file uses git without reimplementing it.

What is urllib used for here?

Stdlib way to download from the internet — here, a .tar.gz of the SDK source from GitHub.

What is a tarball (.tar.gz)?

A compressed archive (like a .zip). GitHub serves a repo at a tag as one; the tool downloads and unpacks it (stripping the single top-level folder GitHub wraps everything in).

git & GitHub
What is git?

A version-control tool: it records every change as a snapshot so you can branch, compare, and revert.

What is a repository?

One project's folder tracked by git (e.g. clevertap-cordova).

What is a branch, and what does the sync create?

An independent line of changes. The sync creates a fresh branch like task/release_<name> so its edits don't touch the base branch until reviewed.

What is the develop / base branch?

The main integration branch the sync starts from and opens its PR against.

What is a commit?

One saved snapshot of changes, with a message.

What is a pull request (PR), and why does it matter here?

A proposal to merge one branch into another, with a diff and a place to review. The whole automation's output is always a PR — never a silent change.

What is a tag, and how does the diff tool use it?

A name pinned to a specific commit, used for releases (e.g. corev8.2.0). The diff tool fetches source at a tag.

What is gh?

GitHub's command-line tool. Scripts use gh pr create, gh label create, etc.

GitHub Actions (the automation runner)
What is GitHub Actions?

GitHub's built-in automation; runs your scripts on GitHub's servers when something happens (a button click, a push, a schedule).

What is a workflow, and where does it live?

A .yml file describing a sequence of automated steps, under .github/workflows/.

What is YAML?

A text format for configuration; indentation matters (like Python). Workflows are written in it.

What is workflow_dispatch?

A trigger meaning "run when a human clicks the button" (a manual dispatch), as opposed to running on a push.

What is workflow_call / a reusable workflow?

A workflow other workflows can call, like a shared function. sync.yml is one; each wrapper repo's tiny dispatch calls it.

What is a composite action? Name a few here.

A reusable bundle of steps (a function within Actions), under .github/actions/. Examples: setup, claude-sync, open-pr, build/*.

What is a runner, and why macos-14 here?

The virtual machine a workflow runs on. The sync uses macos-14 because building the iOS example requires a Mac (Xcode).

What is a step's outcome, and what values can it take?

The status Actions assigns automatically: success, failure, cancelled, or skipped (when its if: was false). Read as steps.<id>.outcome.

What is an if: / gating expression?

A condition on a step deciding whether it runs. The trickiest logic in sync.yml lives here (e.g. "open the PR only if a sync succeeded and nothing was cancelled").

What do always(), failure(), and !cancelled() mean?

always() = run no matter what (even on cancel). failure() = run only if a prior step failed. !cancelled() = run on success or failure, but not if the run was cancelled. Without one, Actions skips remaining steps after any failure.

Why use != 'failure' instead of == 'success' in a gate?

A skipped step (its if: was false, e.g. no iOS version) has outcome skipped, not success. == 'success' would wrongly block a single-platform run; != 'failure' lets skipped through while still blocking a genuine failure.

What is a secret, and how many does this system need?

An encrypted value (API keys, tokens) stored in repo settings that workflows read but humans can't see. This system needs four: App ID, App private key, Anthropic key, Slack webhook URL.

What is a GitHub App here, and what's its name?

A bot identity (clevertap-wrapper-sync) granted permissions to mint short-lived tokens and act on a repo (push a branch, open a PR) without using a person's account.

What is a token, and when is it minted?

A temporary password proving the bot may act. Minted at the start of a run (in Setup); expires when the run ends.

What is an artifact?

A file a workflow saves for download afterward (e.g. the built APK, or Claude's output JSON/stream logs). The sync uploads these if: always() for debugging.

Claude-in-CI (the brain)
What does headless mean, and how is Claude invoked?

Running Claude with no human chatting — given one prompt, it runs to completion on its own. Invoked as claude -p "…" (print mode).

What is a prompt, and where do the big ones live?

The written instructions handed to Claude — effectively a program in English. The big ones live in prompts/ (e.g. sync-orchestrator-cordova.md).

What is .claude/skills/, and how does it differ from the prompt?

Per-repo expert instructions Claude auto-discovers and follows — the real conventions for that wrapper. The prompt is the task; the skills are the know-how.

What is the allowlist (--allowed-tools)?

The exact set of tools Claude may use in a run (e.g. read/edit files, run specific commands). A security boundary — anything not listed is denied. For Cordova, python3 is pinned to only the diff tool.

What is envsubst, and why is it used on the prompt?

A tiny command that fills ${PLACEHOLDER} blanks in a text file with real env values. It turns one static prompt template into a run-specific prompt (injects ${PLATFORM}, ${NEW_VERSION}, etc.).

What is stream-json, and how is the final result extracted?

An output format where Claude emits one JSON event per line (JSON Lines). Scripts use jq 'select(.type=="result")' | tail -1 to pull the final result envelope for cost/token totals.

What is jq?

A command-line tool for slicing JSON. Used to pull the result event out of the stream.

What is the soft cap?

A cost threshold (e.g. $3) that, if exceeded, posts an informational PR comment. It does not stop the run — just visibility.

What is self-heal?

An (optional/retired) retry loop: if a build fails, ask Claude to fix the code and try again, up to N times.

Why does the claude-sync action drop --bare?

--bare would skip auto-discovery of the wrapper repo's .claude/skills. The orchestrator prompt relies on those skills, so running non-bare loads them — a deliberate correctness choice.

Why set +e around the claude call?

So a non-zero exit doesn't instantly abort the script. The action captures the exit code (rc=$?), still writes the result file, prints stderr + last 50 events, then fails deliberately with exit $rc. set -e restores strict mode after.

The 4-layer onion & key concepts
Name the four layers, one phrase each.

① Trigger (a button in a wrapper repo) → ② Conductor (sync.yml sequences & gates) → ③ Brain (Claude, headless, writes the code) → ④ Ground truth (diff_native_api.py, cold facts). Click travels down; PR travels up.

Which layer actually writes the JS/Kotlin/Obj-C bridge code?

Layer ③ — Claude. The conductor (sync.yml) only sequences steps; the YAML wraps no APIs. This is the idea beginners most often get backwards.

Why does a non-AI Python tool (Layer ④) exist if Claude is so capable?

Because an AI can hallucinate a method that doesn't exist. The diff tool reports only what's literally in source, giving Claude a factual anchor it's told to trust and source-verify against.

What are the four stages of diff_native_api.py?

Acquire → Extract → Diff → Render. Get the source, pull out public methods (the surface), compare old vs new, write diff.json + diff.md.

What are the three tiers of acquire_sources, fastest-first?

1) Local clone (if it has the git tag), 2) cache (downloaded on a past run), 3) download the .tar.gz from GitHub. First that works wins.

In _JAVA_PUBLIC_METHOD, what do groups 1/2/3 capture?

Group 1 = return type, group 2 = method name, group 3 = params. Hence return_type, name, params = m.group(1), m.group(2), m.group(3).

What does the negative lookahead (?!class\b|interface\b|…) do?

Ensures the line is NOT a class/interface/enum/constant declaration, so public class CleverTapAPI { isn't mistaken for a method. Keeps the regex to methods only.

Why does the Java extractor do a cheap "public" not in line check first?

Speed. Running regex on every line is wasteful; most lines obviously can't be a public method, so a fast substring test rejects them before the expensive pattern.

What does _is_restricted filter out, and why?

Methods preceded by @RestrictTo/@Hide/@Internal/@VisibleForTesting. They're internal-only and should never be surfaced to wrapper users.

How does compute_diff decide "changed"?

For a name in both versions, it builds the set of signatures in old and in new and compares with !=. If the signature sets differ (param added, overload appeared, etc.), it lands in changed.

Why is everything in compute_diff wrapped in sorted(...)?

Sets are unordered. Sorting makes the output deterministic/reproducible — the same SDK pair always produces byte-identical diff.json, keeping git diffs clean.

For a 8.1.0 → 8.3.0 sync, which changelog entries get read, and why?

The target (8.3.0) plus every version strictly between (8.2.0). Syncs skip versions; reading intermediates surfaces changes that landed in a skipped release.

Why convert version strings to tuples of ints before comparing?

String comparison is wrong ("8.10.0" < "8.9.0" alphabetically). Tuples (8,10,0) vs (8,9,0) compare numerically element-by-element, so ordering and "strictly between" work.

Why does _CHANGELOG_METHOD_CALL require a lowercase first letter?

To skip class names. Methods are lowerCamelCase (recordEvent), classes UpperCamelCase (CleverTapAPI). Anchoring on [a-z] keeps the recall set to likely methods.

What are the THREE Android touches for a new method?

1) CleverTapFunction.java — add the enum constant. 2) CleverTapPlugin.java — add the case in the execute() switch. 3) CleverTapPlugin.java — add the private implementation method.

What are the TWO iOS touches?

The .h declaration and the .m implementation.

Why is a missed Android touch a "silent no-op," and why is that worse than a compile error?

The app still compiles and the method exists, but it never reaches its implementation — runs and does nothing, no crash or warning. A compile error is caught instantly; a silent no-op ships unnoticed. Hence "always all three, then re-verify."

What must be identical across JS, Android, and iOS for one method?

The action string — identical across the JS cordova.exec, the Android enum string, and the iOS selector.

Why does the pre-sync build run BEFORE Claude does anything?

To fail fast and cheap. If the example app is already broken on base_ref, the run stops before spending any AI cost — no point syncing into a pipeline that can't build.

Is a post-sync build failure fatal?

No. The PR still opens with a build-failed label and a warning banner; a human pulls the branch, fixes the compile error, pushes, and merges. ("Option A.")

What is the system's output, every single time?

A pull request for a human to review and merge. The human approval is never automated away.

What does skip_sync=true still run, and what does it skip?

Still runs the pre-sync build (to test the pipeline); skips the AI Sync steps and post-sync build — so $0 AI cost.

What is the "thin pointer" model?

Each wrapper repo carries only a tiny dispatch workflow that calls the hub's sync.yml (plus its .claude/skills). All sync logic lives once in the hub — fix a bug once, every wrapper benefits.

Hub vs. wrapper repos — what lives where?

Hub (clevertap-wrapper-tooling): sync.yml, composite actions, diff_native_api.py, prompts/, scripts/. Wrapper (clevertap-cordova etc.): the tiny native-release-sync.yml dispatch, .claude/skills/, and the bridge code + example apps the automation edits.
