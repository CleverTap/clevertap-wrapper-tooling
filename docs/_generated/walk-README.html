Walkthroughs — read the system one file at a time

This folder is the line-by-line tour of the CleverTap native-release-sync automation. Each page takes one real source file (or one section of it) and explains what every line does, visually, for someone who has never written Python, YAML, shell, or a GitHub Action.

Mental model

How to use this folder: start at the top of the ranking table and work down. The 4-layer onion primer is the map; these pages are the street-level walk. Every new word is defined inline in a sidebar and again in the GLOSSARY.

What gets line-by-line vs annotated-highlights, and why

Not every file deserves the same microscope. Tricky logic (regex, gating if: expressions, shell exit handling) gets every line. Configuration that is mostly "fill in the blank" gets annotated highlights — we point at the lines that matter and skip the boilerplate.
